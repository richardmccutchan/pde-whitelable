The Perks

Rapid prototyping and development: Kitchen Sink combines Skeleton's neat and efficient gridsystem with the most important UI elements, effortless font import, a few helper classes and normalize. If you use this page as a starting template, you'll even see modernizr and jQuery CDNs in place.

Design Consistency: We are a design-oriented company. Having a solid foundation assures that websites look good across devices and remain in line with brand identity.

Without having written a single line of code, Kitchen Sink provides you with an end-to-end responsive experience. And you guessed it: We're using Sass to make customisation as beautiful as possible. Not that you needed to, since all brand colors are pre-defined as accessible variables.

This page is the kitchen sink of Kitchen Sink (wow, so meta). So if there is a specific element you are looking for, just look under the hood.

Typography

Fonts

Getting fonts right on the web is tricky. Thankfully, as part of Kitchen Sink, importing fonts is breezy-easy.

How easy? I am glad you asked: You literally don't have to do anything. All the fonts we use for web projects come bundled with Kitchen Sink. However, to improve efficiency and loading times delete font files you don't need and remove their respective .scss mixin.


            /* example: class that requires heavy font */
            .heavy {
            font-family: "Corpid C1s Heavy", "Corpid C1s", Helvetica, Arial, sans-serif;
            /* Helvetica and Arial as web-safe fallbacks */
            font-weight: 900;
            /* define font-weight for fallbacks */
          }

We all know how awesome buttons are. Even the word button is cute. As illustrated below, there are five ways to create a button. While the .button class will also work on <span> or <div>, elements with inherent interactivity should behave and look like a button.

Contextual Buttons

Using the appropriate button in the right context is fundamental to facilitating user interaction. Different colors trigger different learned behaviour. For instance, .button-confirm is a typical example of a positive submit button, whereas .button-delete is used in a destructive context, like cancel or delete.

.button-cta is used primarily to invoke user interaction because apparentely this company gave up on an aesthetic vision in favour of well-performing but hideous designs.
